# 任务规划 Skill

用于将复杂任务分解为可执行的子任务序列，并分配至合适的 Agent。

## 使用场景

- 遇到大型功能开发需求时
- 需要多个 Agent 协作的复杂任务
- 需要制定阶段性交付计划时

## 工作流程

1. **需求分析** — 理解用户需求的完整范围
2. **任务分解** — 使用 WBS（工作分解结构）方法拆解
3. **依赖分析** — 确定任务间的执行顺序和依赖关系
4. **资源分配** — 为每个子任务分配最合适的 Agent
5. **时间估算** — 评估每个子任务的工作量

## 输出格式

```markdown
## 任务计划

### 阶段 1：[阶段名称]
- 任务 1.1：[描述] → @agent_name
- 任务 1.2：[描述] → @agent_name

### 阶段 2：[阶段名称]
- 任务 2.1：[描述] → @agent_name

### 依赖关系
- 任务 1.1 → 任务 1.2
- 任务 1.2 → 任务 2.1

### 交付标准
- [明确的可验收标准]
```
