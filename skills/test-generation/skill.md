# 测试生成 Skill

用于为代码自动生成测试用例，包括单元测试和集成测试。

## 使用场景

- 新功能代码完成时
- 发现未覆盖的代码路径时
- 修复 Bug 后添加回归测试

## 测试生成策略

1. **接口分析** — 分析函数的输入/输出和边界条件
2. **路径覆盖** — 确保覆盖所有主要执行路径
3. **Mock 策略** — 确定需要模拟的外部依赖
4. **Fixture 设计** — 准备测试数据和夹具

## 覆盖目标

- 语句覆盖：≥ 90%
- 分支覆盖：≥ 85%
- 函数覆盖：100%（公开 API）

## 调用方式

```
@tester 请使用 test-generation 技能为 [文件路径] 生成测试
```
