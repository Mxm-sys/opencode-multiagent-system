# 代码审查工作流

## 触发条件
- Developer 完成功能实现并提交代码
- Reviewer 收到审查请求

## 审查流程

```
提交代码 → 自动检查 → 人工审查 → 反馈修改 → 确认通过
```

### 步骤 1：自动检查
- 运行 Lint 和类型检查
- 运行已有测试套件
- 检查代码格式规范

### 步骤 2：代码审查 (Reviewer)
- 按审查清单逐项检查
- 给出严重程度评级
- 提供修改建议

### 步骤 3：反馈修改
- 将审查结果反馈给 Developer
- Developer 按建议修改
- 标记已解决的问题

### 步骤 4：最终确认
- Reviewer 确认所有问题已解决
- 输出审查通过结论

## 审查标准

### 严重程度定义
- **严重**：功能错误、安全漏洞、性能灾难 — 必须修复
- **中等**：代码异味、可维护性问题 — 建议修复
- **轻微**：格式、命名风格 — 可选修复

### 审查通过条件
1. 无严重级别问题
2. 中等问题不超过 3 个
3. 所有测试通过
4. 代码风格一致
